Contextualização

Num mercado como o de hoje, em que ser-se competitivo é quase equivalente a sobreviver, é fundamental sobrepormos as restrições que nos são postas e procurar e investir em mercados com necessidades específicas.

Por vezes as instituições focam-se em criar soluções exclusivas para o grande mercado genérico, negligenciando necessidades mais específicas e é neste contexto microaplicacional que se vem propor este projecto, isto é, sendo possível desenvolver uma componente que interopere com o panorama geral e que trabalhe com essas necessidades específicas, podem daí resultar pequenos produtos com uma economia de escala associada.

Assim sendo, fixou-se um tema que nasceu no ceio de conversas e vivências de um dos docentes: uma escola de música. Existem actualmente algumas dezenas de aplicações para a gestão de escolas mas nenhuma se foca nas especificidades de uma escola de música. Para tal, identificaram-se algumas necessidades específicas através de uma reunião com a direcção de uma escola de música, necessidades essas que poderão ser solucionadas por microaplicações: um gestor de audições/concertos e um gestor de programas específicos.

Funcionalidades

A aplicação criada neste projeto permite aos utilizadores:

  • Criar uma nova audição a partir de informação fornecidapelo utilizador através de um ficheiro;
  • Alterar metainformação relativa a uma audição que ainda não tenha ocorrido;
  • Consultar toda a informação relativa a uma audição;
  • Consultar a lista de todas as Audições;
  • Ver as últimas audições criadas;
  • Inserir Alunos, Professores, Cursos e Obras utilizando um formulário no site ou um ficheiro XML com os dados;
  • Consultar toda a informação relativa a Alunos, Professores, Cursos e Obras;
  • Alterar Alunos, Professores, Cursos e Obras;
  • Remover Alunos, Professores, Cursos e Obras;
  • Verificar algumas estatísticas sobre a informação contida na base de dados.

Implementação

As operações no site são feitas através de formulários atendidos por várias scripts PHP para fazer a consulta à base de dados.

Para a criação das audições, o site comunica com um servlet enviando um ficheiro escrito numa linguagem própria. O servlet invoca o ANTLR para fazer a análise do ficheiro, inserindo a informação referente à audição na base de dados. Após a inserção o servlet responde ao cliente através de JSON com o resultado do parse/inserção.

Gramática

Uma audição é criada através de uma ficheiro escrito numa linguagem própria. Para tal desenvolvemos a seguinte gramática:

s : metadata atuacoes ;
metadata : 'METADATA' audId titulo subtitulo tema data hora local organizador duracao ;
titulo : 'TITULO' DADOS ;
subtitulo : 'SUBTITULO' DADOS ;
tema : 'TEMA' DADOS ;
data : 'DATA' INT '-' INT '-' INT ;
hora : 'HORA' DADOS ;
local : 'LOCAL' DADOS ;
organizador : 'ORGANIZADOR' DADOS ;
duracao : 'DURACAO' DADOS ;
atuacoes : 'ATUACOES' '{' (atuacao)+ ’'}' ;
atuacao : 'ATUACAO' id designacao alunos professores? pecas;
designacao : 'DESIGNACAO' DADOS ;
alunos : 'ALUNOS' '{' aluno (',' aluno)* '}' ;
aluno : id ;
audId : 'ID' ID ;
id : ID ;
professores : 'PROFESSORES' '{' professor (',' professor])* '}' ;
professor : id ;
pecas : 'PECAS' '{' peca (',' peca)* '}' ;
peca : id ;

Conclusão

Concluído o projeto verificamos que pudemos aplicar os conhecimentos adquiridos nas cadeiras de Processamento e Representação de Informação (PRI) e Gramáticas na Compreensão de Software (GCS) para criar uma aplicação que permite providenciar ao utilizador diferentes funcionalidades.
